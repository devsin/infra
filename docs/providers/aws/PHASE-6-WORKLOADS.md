# Phase 6: Workloads (GitOps)

## 🎯 Objective

Deploy application workloads using GitOps principles with:
- Argo CD app-of-apps pattern
- Kustomize base/overlays per environment
- External Secrets for secret management
- CI/CD pipeline for build → push → deploy
- Argo CD Image Updater for automatic rollouts

## 🏗️ Architecture

```
Developer ──► GitHub ──► GitHub Actions (CI) ──► ECR (Image)
                                                      │
                                                      ▼
              Argo CD ◄── Git Repo (manifests) ◄── Image Updater
                 │
                 ▼
              EKS Cluster ──► Running Pods
```

## 📁 GitOps Repo Structure

> **Brand-named folders _are_ required here** — Kustomize and Argo CD need real directory paths.
> But you never create them manually. A scaffolding script reads your `brands.yaml`
> config and generates the entire structure.

### brands.yaml (single source of truth)

```yaml
brands:
  - name: acme
    domain: acme.com
    services: [api, web, worker]
    envs: [dev, stage, prod]
  - name: widgets
    domain: widgets.io
    services: [api, web]
    envs: [dev, prod]
```

### Scaffolding Script

```bash
#!/usr/bin/env bash
# scripts/scaffold.sh — reads brands.yaml and creates the GitOps directory tree
# Requires: yq (https://github.com/mikefarah/yq)

set -euo pipefail

for brand in $(yq '.brands[].name' brands.yaml); do
  envs=$(yq ".brands[] | select(.name == \"$brand\") | .envs[]" brands.yaml)
  services=$(yq ".brands[] | select(.name == \"$brand\") | .services[]" brands.yaml)

  for env in $envs; do
    mkdir -p "apps/$brand"
    # Generate Argo CD Application YAML for this brand/env
    cat > "apps/$brand/$env.yaml" <<EOF
apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: "$brand-$env"
  namespace: argocd
  labels:
    brand: "$brand"
    environment: "$env"
spec:
  project: default
  source:
    repoURL: "REPLACE_WITH_REPO_URL"
    targetRevision: main
    path: "workloads/$brand"
    directory:
      recurse: true
      include: "*/overlays/$env/*"
  destination:
    server: https://kubernetes.default.svc
    namespace: "$brand"
  syncPolicy:
    automated:
      prune: true
      selfHeal: true
    syncOptions:
      - CreateNamespace=true
EOF
  done

  for svc in $services; do
    mkdir -p "workloads/$brand/$svc/base"
    for env in $envs; do
      mkdir -p "workloads/$brand/$svc/overlays/$env"
    done
  done

  mkdir -p platform/namespaces
done

echo "✅ Scaffold complete. Now fill in base/ manifests for each service."
```

> **To add a new brand:** edit `brands.yaml`, re-run `./scripts/scaffold.sh`. No Terraform changes needed.

## 🔄 CI/CD Pipeline

### .github/workflows/build-and-push.yaml

```yaml
name: Build and Push

on:
  push:
    branches: [main]
    paths:
      - "src/**"
      - "Dockerfile"

env:
  AWS_REGION: "eu-west-1"  # Set to your primary_region

permissions:
  id-token: write
  contents: read

jobs:
  build:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        # Define your brands and their account IDs here
        # These are the only places brand names appear in CI
        include:
          - brand: acme
            account_id: "111111111111"
          - brand: widgets
            account_id: "222222222222"

    steps:
      - uses: actions/checkout@v4

      - name: Configure AWS credentials
        uses: aws-actions/configure-aws-credentials@v4
        with:
          role-to-assume: "arn:aws:iam::${{ matrix.account_id }}:role/${{ vars.PREFIX }}-automation"
          aws-region: ${{ env.AWS_REGION }}

      - name: Login to ECR
        id: ecr
        uses: aws-actions/amazon-ecr-login@v2

      - name: Build and push
        uses: docker/build-push-action@v5
        with:
          context: .
          push: true
          tags: |
            ${{ steps.ecr.outputs.registry }}/${{ matrix.brand }}/api:${{ github.sha }}
            ${{ steps.ecr.outputs.registry }}/${{ matrix.brand }}/api:latest
```

## 🔄 Deployment Flow

```
1. Developer pushes code to main branch
          │
          ▼
2. GitHub Actions CI triggers
   • Build Docker image
   • Push to ECR (<brand>/api:SHA)
          │
          ▼
3. Argo CD Image Updater detects new image
   • Updates Git manifests with new tag
          │
          ▼
4. Argo CD detects Git change
   • Syncs manifests to EKS cluster
          │
          ▼
5. Kubernetes rolls out new version
   • Rolling update with zero downtime
          │
          ▼
6. Health checks pass → Traffic shifts
```

## 🚀 Deployment Steps

### 1. Set up GitOps repository

```bash
# Create the gitops manifests repo
mkdir my-gitops-manifests
cd my-gitops-manifests
git init

# Define your brands in brands.yaml (see above)
cp /path/to/brands.yaml .
cp /path/to/scripts/scaffold.sh scripts/

# Generate the entire directory structure
chmod +x scripts/scaffold.sh
./scripts/scaffold.sh

# Commit and push
git add .
git commit -m "Initial gitops structure"
git remote add origin https://github.com/<your-org>/<your-repo>.git
git push -u origin main
```

### 2. Connect Argo CD to the repo

```bash
# Add the repo to Argo CD
argocd repo add https://github.com/<your-org>/<your-repo>.git \
  --username git \
  --password $GITHUB_TOKEN

# Create the app-of-apps for each brand/environment
# (these YAML files were generated by scaffold.sh)
kubectl apply -f apps/
```

### 3. Install Argo CD Image Updater

```bash
# Install via Helm (if not already in Phase 5 add-ons)
helm repo add argo https://argoproj.github.io/argo-helm
helm install argocd-image-updater argo/argocd-image-updater \
  --namespace argocd \
  --set config.registries[0].name=ECR \
  --set config.registries[0].prefix=ACCOUNT_ID.dkr.ecr.REGION.amazonaws.com \
  --set config.registries[0].api_url=https://ACCOUNT_ID.dkr.ecr.REGION.amazonaws.com \
  --set config.registries[0].credentials=ext:/scripts/ecr-login.sh
```
